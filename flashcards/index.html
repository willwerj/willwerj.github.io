CSV Format Guide

Overview

Flashcard sets are created from a CSV (comma-separated values) file. Each row becomes one flashcard. The first row must be a header row with exact column names.

Required Columns

Your CSV must include all seven of these columns (in any order):

ColumnDescription
questionThe front of the card (what is shown first)
answerThe back of the card (revealed on flip)
card_typeOne of: definition, true_false, conceptual
concept_tagThe named concept this card relates to (e.g. "Entropy")
chapterThe chapter or section this card belongs to
source_fileFilename or URL of the original source material
page_numberPage number in the source (or N/A)

LaTeX Support

Math expressions are supported using LaTeX notation. Wrap inline math in $…$ and display math in $$…$$. Example: $P(k) = \frac{e^{-\lambda}\lambda^k}{k!}$

Important Formatting Rules

  • If a field contains a comma, wrap the entire field in double quotes: "mean, variance"
  • If a field contains a double quote, escape it by doubling it: "He said ""hello"""
  • Do not leave the question or answer fields empty
  • Column names are case-insensitive but must match exactly otherwise

Example

chapter,concept_tag,card_type,question,answer,source_file,page_number Chapter 1,Entropy,definition,What is entropy?,The average uncertainty of a distribution.,notes.pdf,5 Chapter 1,Binomial,true_false,"True or False: $\binom{n}{k}$ counts sequences with k heads.","True. It counts the number of ways to choose k slots out of n.",notes.pdf,8 Chapter 2,Fokker-Planck,conceptual,"What does the Fokker-Planck equation govern?","It governs how a probability density $\varrho(x,t)$ evolves under an SDE.",notes.pdf,12

Common Errors

  • "Missing required columns" — Check spelling and case of your header row
  • "Could not parse CSV" — Usually means unescaped commas or quotes. Wrap the field in double quotes
  • "No data rows found" — Your file only has a header row and no card data
